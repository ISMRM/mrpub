What can be shared on MR-Pub?

MR-Pub currently welcomes interactive Jupyter Books along two tracks:

  • Education notebooks related to a workshop in the MRI field.
  • Research notebooks for articles published in Magnetic Resonance in Medicine.

How to submit?

  1. Create a public repository on GitHub which will host the files.
  2. Create a content folder at the root of your repository:
  • Place your Jupyter notebook(s) (.ipynb) and markdown files (optional)
  • Create a _toc.yml to structure your book with the table of contents.
  • Create a _config.yml to configure book settings.
  1. Create a binder folder at the root of your repository:
  • Add your configuration files to capture all the software dependencies.
  • For a list of sample repositories for use with Binder, see the Sample Binder Repositories page.
  1. Go to https://mybinder.org
  2. Fill the repository name and save the given URL. This will be used to share your binder notebooks with others.
  3. Click on launch and wait for the build. Please be patient, as the first build may take a long time depending on your setup. When your notebook loads, you are ready to share your notebooks!
  4. Create an issue in this repository, using the “Submission” template, and fill in all the information.
  5. Our team will fork your repository on https://github.com/neurolibre. We will test that everything is working, and contact you using your github handle for any issue we identify. The issues will be open on our fork of the repo, and you will be expected to submit pull request to this fork to address any issue.
